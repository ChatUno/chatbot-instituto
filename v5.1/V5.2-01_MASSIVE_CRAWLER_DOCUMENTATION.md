# V5.2-01 Massive Crawler Documentation

## Overview

The V5.2-01 Massive Crawler is an intelligent website crawling system designed to process entire websites efficiently and responsibly. It provides advanced features including adaptive rate limiting, duplicate detection, real-time monitoring, and comprehensive reporting.

## Features

### Core Functionality
- **Multi-threaded Crawling**: Concurrent processing of multiple pages
- **Intelligent Link Discovery**: Advanced URL extraction and filtering
- **Adaptive Rate Limiting**: Automatic adjustment based on server responses
- **Duplicate Detection**: Content and URL deduplication
- **State Persistence**: Resume crawling after interruptions
- **Real-time Monitoring**: Live progress tracking and statistics

### Advanced Features
- **Retry System**: Exponential backoff for failed requests
- **Queue Management**: Efficient processing of large websites
- **Comprehensive Reporting**: Detailed statistics and export options
- **Configurable Filters**: Customizable inclusion/exclusion patterns
- **Error Handling**: Robust error recovery and logging

## Architecture

### Backend Services

#### MassiveCrawler
The main crawling engine that orchestrates the entire process.

**Key Methods:**
- `initialize()`: Initialize browser and crawling components
- `crawl(baseUrl)`: Start crawling from a base URL
- `processQueue()`: Manage concurrent page processing
- `extractLinks(html, baseUrl, domain, depth)`: Extract and filter links
- `savePageData(pageData)`: Store page content and metadata

**Configuration Options:**
```javascript
{
  maxConcurrency: 5,        // Maximum parallel requests
  maxDepth: 3,             // Maximum link depth
  rateLimitDelay: 1000,     // Base delay between requests
  userAgent: '...',          // Custom user agent
  timeout: 30000,           // Request timeout
  retryAttempts: 3,         // Number of retry attempts
  retryDelay: 5000          // Base retry delay
}
```

#### QueueManager
Manages the processing queue for large-scale crawling operations.

**Key Methods:**
- `enqueue(items)`: Add items to processing queue
- `getNextBatch(maxItems)`: Get next batch for processing
- `markCompleted(itemIds)`: Mark items as completed
- `markFailed(itemIds, error)`: Mark items as failed
- `getQueueStats()`: Get queue statistics

#### RateLimiter
Implements adaptive rate limiting per domain.

**Key Methods:**
- `wait(domain)`: Wait before making request
- `recordSuccess(domain, responseTime)`: Record successful request
- `recordError(domain, error)`: Record failed request
- `setDomainDelay(domain, delay)`: Manually set domain delay
- `getDomainStats(domain)`: Get domain-specific statistics

#### LinkDiscovery
Intelligent link extraction and filtering system.

**Key Methods:**
- `discoverLinks(html, baseUrl, currentDepth, maxDepth)`: Discover links
- `filterLinks(links, targetDomain)`: Filter discovered links
- `deduplicateLinks(links)`: Remove duplicate links
- `categorizeLinks(links)`: Categorize links by type

#### DuplicateDetection
Advanced duplicate detection using multiple strategies.

**Key Methods:**
- `checkDuplicate(pageData)`: Check if page is duplicate
- `findSimilarPages(content, threshold)`: Find similar pages
- `getDuplicateGroups()`: Get groups of duplicate pages
- `clearCache()`: Clear duplicate detection cache

#### StatePersistence
Persistent storage for crawling state and progress.

**Key Methods:**
- `saveState()`: Save current crawling state
- `loadState()`: Load saved crawling state
- `createBackup(label)`: Create state backup
- `restoreBackup(backupFile)`: Restore from backup

#### DiscoveryReporting
Comprehensive reporting and analytics system.

**Key Methods:**
- `initialize(domain)`: Initialize reporting for domain
- `addPageDiscovered(pageData)`: Record discovered page
- `addPageProcessed(pageData)`: Record processed page
- `addPageFailed(pageData, error)`: Record failed page
- `generateSummary()`: Generate crawling summary
- `exportReport(format)`: Export report in various formats

## API Endpoints

### Crawling Management

#### Start Crawling
```
POST /api/massive-crawler/start
Content-Type: application/json

{
  "url": "https://example.com",
  "options": {
    "crawler": { ... },
    "queue": { ... },
    "discovery": { ... },
    "rateLimiter": { ... },
    "retry": { ... },
    "duplicate": { ... },
    "reporting": { ... }
  }
}
```

**Response:**
```json
{
  "success": true,
  "sessionId": "crawl_1234567890_abc123",
  "status": "started",
  "url": "https://example.com",
  "message": "Massive crawling started successfully"
}
```

#### Get Crawling Status
```
GET /api/massive-crawler/status/:sessionId
```

**Response:**
```json
{
  "success": true,
  "sessionId": "crawl_1234567890_abc123",
  "status": "running",
  "startTime": "2023-12-01T10:00:00.000Z",
  "stats": {
    "totalDiscovered": 150,
    "totalProcessed": 120,
    "totalFailed": 5,
    "processingRate": 45.2
  },
  "queueStats": {
    "pending": 25,
    "processing": 3,
    "completed": 120,
    "failed": 5
  },
  "reportStats": {
    "categories": {
      "homepage": 1,
      "navigation": 15,
      "content": 80,
      "documents": 20,
      "media": 4
    }
  }
}
```

#### Stop Crawling
```
POST /api/massive-crawler/stop/:sessionId
```

#### Delete Session
```
DELETE /api/massive-crawler/session/:sessionId
```

### Configuration Management

#### Update Configuration
```
PUT /api/massive-crawler/config/:sessionId
Content-Type: application/json

{
  "config": {
    "rateLimiter": {
      "defaultDelay": 2000,
      "maxDelay": 20000
    },
    "retry": {
      "maxAttempts": 5
    }
  }
}
```

#### Set Domain Rate Limit
```
POST /api/massive-crawler/rate-limit/:sessionId/:domain
Content-Type: application/json

{
  "delay": 5000
}
```

### Data Access

#### Get Pages
```
GET /api/massive-crawler/pages/:sessionId?page=1&limit=50&status=processed&category=content
```

#### Get Errors
```
GET /api/massive-crawler/errors/:sessionId?page=1&limit=50
```

#### Get Discovery Report
```
GET /api/massive-crawler/report/:sessionId?format=json
```

Supported formats: `json`, `csv`, `html`

#### Get Statistics
```
GET /api/massive-crawler/stats/:sessionId
```

#### List Sessions
```
GET /api/massive-crawler/sessions
```

## Configuration Options

### Crawler Configuration
```javascript
{
  maxConcurrency: 5,        // Max parallel requests (1-20)
  maxDepth: 3,             // Max link depth (1-10)
  rateLimitDelay: 1000,     // Base delay in ms (100-10000)
  userAgent: '...',          // Custom user agent string
  timeout: 30000,           // Request timeout in ms (5000-120000)
  retryAttempts: 3,         // Retry attempts (0-10)
  retryDelay: 5000          // Retry delay in ms (1000-30000)
}
```

### Queue Configuration
```javascript
{
  maxQueueSize: 10000,      // Max queue size (100-100000)
  batchSize: 100            // Processing batch size (10-1000)
}
```

### Discovery Configuration
```javascript
{
  maxLinksPerPage: 200,     // Max links per page (10-1000)
  excludePatterns: [        // Regex patterns to exclude
    /\.(pdf|doc|docx)$/i,
    /\/api\//i,
    /\/admin\//i
  ]
}
```

### Rate Limiter Configuration
```javascript
{
  defaultDelay: 1000,       // Default delay in ms
  maxDelay: 10000,          // Maximum delay in ms
  minDelay: 100,            // Minimum delay in ms
  adaptiveMode: true,        // Enable adaptive rate limiting
  errorThreshold: 3,         // Errors before slowing down
  successThreshold: 10        // Successes before speeding up
}
```

### Retry Configuration
```javascript
{
  maxAttempts: 3,           // Maximum retry attempts
  baseDelay: 1000,          // Base retry delay in ms
  maxDelay: 30000,          // Maximum retry delay in ms
  backoffFactor: 2,         // Exponential backoff factor
  jitter: true               // Add random jitter to delays
}
```

### Duplicate Detection Configuration
```javascript
{
  similarityThreshold: 0.85,  // Similarity threshold (0.1-1.0)
  maxCacheSize: 100000,       // Maximum cache size
  enableContentHashing: true,   // Enable content-based detection
  enableUrlNormalization: true, // Enable URL normalization
  enableTitleMatching: true     // Enable title-based detection
}
```

### Reporting Configuration
```javascript
{
  realTimeUpdates: true,      // Enable real-time updates
  includeScreenshots: false,   // Include screenshots in reports
  maxReportHistory: 50        // Maximum report history
}
```

## Usage Examples

### Basic Crawling
```javascript
// Start crawling
const response = await fetch('/api/massive-crawler/start', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    url: 'https://example.com',
    options: {
      crawler: { maxConcurrency: 3, maxDepth: 2 }
    }
  })
});

const { sessionId } = await response.json();

// Monitor progress
const statusResponse = await fetch(`/api/massive-crawler/status/${sessionId}`);
const status = await statusResponse.json();
console.log('Progress:', status.stats);
```

### Advanced Configuration
```javascript
const config = {
  crawler: {
    maxConcurrency: 8,
    maxDepth: 4,
    rateLimitDelay: 500,
    timeout: 60000
  },
  discovery: {
    maxLinksPerPage: 300,
    excludePatterns: [
      /\.(pdf|doc|docx|xls|xlsx)$/i,
      /\/api\//i,
      /\/admin\//i,
      /\/login/i
    ]
  },
  rateLimiter: {
    defaultDelay: 500,
    maxDelay: 15000,
    adaptiveMode: true,
    errorThreshold: 2
  },
  duplicate: {
    similarityThreshold: 0.9,
    enableContentHashing: true,
    enableUrlNormalization: true
  }
};

const response = await fetch('/api/massive-crawler/start', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    url: 'https://example.com',
    options: config
  })
});
```

### Real-time Monitoring
```javascript
const sessionId = 'crawl_1234567890_abc123';

// Set up real-time updates
const monitorSession = async () => {
  const response = await fetch(`/api/massive-crawler/status/${sessionId}`);
  const data = await response.json();
  
  console.log(`Progress: ${data.stats.totalProcessed}/${data.stats.totalDiscovered}`);
  console.log(`Processing rate: ${data.stats.processingRate} pages/min`);
  console.log(`Queue size: ${data.queueStats.pending}`);
  
  if (data.status === 'completed' || data.status === 'failed') {
    console.log('Crawling finished:', data.status);
    return;
  }
  
  // Continue monitoring
  setTimeout(monitorSession, 5000);
};

monitorSession();
```

## Error Handling

### Common Error Codes
- `MISSING_URL`: URL not provided in start request
- `INVALID_URL`: URL format is invalid
- `SESSION_NOT_FOUND`: Session ID does not exist
- `START_FAILED`: Failed to start crawling
- `STOP_FAILED`: Failed to stop crawling
- `CONFIG_UPDATE_FAILED`: Failed to update configuration

### Error Response Format
```json
{
  "success": false,
  "error": "Error message",
  "code": "ERROR_CODE",
  "details": "Additional error details"
}
```

## Troubleshooting

### Common Issues

#### Crawling Stops Unexpectedly
- Check rate limiting configuration
- Verify domain accessibility
- Review error logs for patterns
- Check memory usage

#### High Memory Usage
- Reduce max concurrency
- Lower queue size limits
- Clear duplicate detection cache
- Reduce max depth

#### Slow Processing
- Adjust rate limiting delays
- Increase concurrency (within limits)
- Check network connectivity
- Review exclude patterns

#### Duplicate Detection Issues
- Adjust similarity threshold
- Enable/disable different detection methods
- Clear duplicate cache
- Review URL normalization settings

## Best Practices

### Responsible Crawling
1. **Always respect robots.txt**: The system automatically checks and respects robots.txt files
2. **Use appropriate delays**: Start with conservative rate limits and adjust based on server response
3. **Monitor error rates**: High error rates may indicate aggressive crawling
4. **Set reasonable depth limits**: Avoid infinite crawling loops
5. **Regular backups**: Use state persistence to prevent data loss

### Performance Optimization
1. **Tune concurrency**: Balance between speed and server load
2. **Optimize filters**: Effective exclude patterns reduce unnecessary processing
3. **Monitor resources**: Track memory and CPU usage
4. **Use batch processing**: Optimize queue batch sizes for your use case
5. **Regular cleanup**: Clear old data and caches periodically

### Data Management
1. **Regular backups**: Schedule automatic state backups
2. **Export reports**: Save crawling results in preferred formats
3. **Monitor storage**: Watch disk usage for large crawling jobs
4. **Archive old data**: Move completed crawls to long-term storage
